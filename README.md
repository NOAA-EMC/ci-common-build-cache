# ci-common-build-cache

This repository contains a GitHub Actions CI workflow that uses Spack to generate a GitHub Packages cache of pre-built dependency packages for various EMC codes, as well as unified container images for a number of applications, compilers, and MPI libraries.

For an overview of the workflow and instructions for maintenance, see the [repo wiki](https://github.com/NOAA-EMC/ci-common-build-cache/wiki).

## Using the Spack build cache

To use the Spack build cache generated by this workflow, add the URL for the Spack mirror and trust the GPG signing key:
```console
$ spack mirror add emc-common-build-cache oci://ghcr.io/NOAA-EMC/ci-common-build-cache-{gcc,oneapi}
$ wget https://github.com/NOAA-EMC/ci-common-build-cache/releases/download/may2026-spack-key/spack-signing-public.asc # immutable release; see also https://keys.openpgp.org/search?q=alexander.richert%40noaa.gov
$ spack gpg trust spack-signing-public.asc
```

Trusting the GPG key enables signature verification of packages from the build cache. Without this step, you will need to use `--no-check-signature` when installing from the cache.

### Requirements

Any GitHub Actions CI workflow using this Spack build cache must provide the following:
- Spack (to concretize the environment and access the build cache)
- Compilers (GCC, Intel Classic, Intel OneAPI; see .github/workflows/build.yaml for available versions; use apt or https://github.com/NOAA-EMC/ci-install-intel-toolkit to install)
- Intel MPI (see .github/workflows/build.yaml for available versions; use apt or https://github.com/NOAA-EMC/ci-install-intel-toolkit to install)

Note that the [NOAA-EMC/ci-test-spack-package](https://github.com/NOAA-EMC/ci-test-spack-package) custom GitHub Action automatically enables this build cache by default, and can be used to install a package (i.e., modified code being tested in CI) and its dependencies, or dependencies only.

## Using container images

To use a container image provided by this repository's GitHub Packages registry:
1. Go to https://github.com/orgs/NOAA-EMC/packages?repo_name=ci-common-build-cache
2. Select a container image name based on the application and configuration to be tested, e.g., `nceplibs-numbered-ubuntu-24.04-gcc-13-mpich-x`.
3. Click on "View and manage all versions" to select a specific version by SHA256 digest *or* click on "latest" to use the most recent version of that image.
 - **To run the image in a GitHub Actions workflow**: use the `container` or `container:image` attribute to point to the URL (i.e., the above information without the `docker` command). See GitHub's documentation on [Running jobs in a container](https://docs.github.com/en/actions/how-tos/write-workflows/choose-where-workflows-run/run-jobs-in-a-container).
 - **To pull and run the image locally**: Copy and paste the command for pulling the image, e.g., `docker pull ghcr.io/noaa-emc/ci-common-build-cache/nceplibs-numbered-ubuntu-24.04-gcc-13-mpich-x@sha256:ac842ac75e0b05b5f242317902765162c3cf4442dc74f5b3150bfa0990883407`. To run the image interactively: `docker run -it docker pull ghcr.io/noaa-emc/ci-common-build-cache/nceplibs-numbered-ubuntu-24.04-gcc-13-mpich-x@sha256:ac842ac75e0b05b5f242317902765162c3cf4442dc74f5b3150bfa0990883407`. This may require logging into GitHub: https://docs.github.com/en/packages/working-with-a-github-packages-registry/working-with-the-container-registry.

See the templates in the applications/ directory for the packages, versions, and build options for each container image.

To verify a container key with `cosign`:
```console
wget https://github.com/NOAA-EMC/ci-common-build-cache/releases/download/may2026-cosign-key/cosign.pub # immutable release
cosign verify --key cosign.pub ghcr.io/noaa-emc/ci-common-build-cache/gsi-ubuntu-24.04-gcc-13-mpich-x:latest
```
